📋 Root Cause Analysis

The original detection engine contained 10 critical bugs that caused malicious URLs to be classified as safe. Each bug is documented below with the exact code change applied.

10 Bugs Fixed 14 Detection Checks Levenshtein Typosquatting Homograph Detection Brand Impersonation
🐛 Bugs Found & Fixed
BUG 01
Blacklist check returned score 0 on match. The blacklist was queried but the result was discarded — malicious domains were detected then immediately scored as safe.
Fix: Return +50 pts on exact match, +45 pts on parent-domain match.
❌ Before (broken)
if (BLACKLISTED.contains(host)) {
  threats.add("BLACKLISTED");
  return 0; // ← BUG: always safe!
}
✅ After (fixed)
if (BLACKLISTED.contains(host)) {
  threats.add("BLACKLISTED");
  return 50; // ← high severity
}
BUG 02
IP-address host check returned 0 pts. URLs like http://192.168.1.1/login were marked safe.
Fix: Raw IP hosts score +40 pts; private-range IPs score +35 pts.
BUG 03
SSL check was inverted. HTTP (no encryption) was flagged as safe; HTTPS was flagged as suspicious.
Fix: HTTP adds +15 pts (risk). HTTPS adds 0 pts (expected baseline).
❌ Before (broken)
if (scheme.equals("https")) {
  return 20; // ← treated HTTPS as threat!
} else {
  return 0;
}
✅ After (fixed)
if (scheme.equals("http")) {
  return 15; // ← HTTP is the risk
} else {
  return 0; // HTTPS = baseline ok
}
BUG 04
Typosquatting threshold too loose (Levenshtein ≤ 5). A domain 5 characters away from "paypal.com" is not necessarily spoofing it — this created false positives while missing actual typosquats.
Fix: Distance ≤ 2 = TYPOSQUATTING (+40 pts). Distance > 2 = BRAND_IMPERSONATION (+30 pts).
BUG 05
Phishing keyword check added 0 to score on match. Keywords like "login", "verify", "secure" were matched but the returned value was always 0.
Fix: Weighted scoring: 1 keyword=10 pts, 2=20, 3=28, 4+=capped at 35.
BUG 06
URL shorteners were classified as SAFE. bit.ly, tinyurl.com etc. hide the destination entirely — the engine was returning SAFE without resolving redirects.
Fix: Shorteners score +25 pts with MASKED_DESTINATION threat indicator.
BUG 07
Newly registered domain check was entirely absent. Phishing sites registered hours before an attack were scored 0 on this dimension.
Fix: Heuristic: free TLD + phishing keyword in domain + brand abuse = +20 pts + NEW_PHISHING_DOMAIN threat.
BUG 08
Homograph / IDN attack detection was absent. URLs with Unicode lookalike characters (e.g. microsоft.com with Cyrillic 'о') were undetected.
Fix: Non-ASCII characters in hostname score +45 pts (HOMOGRAPH_ATTACK).
BUG 09
Verdict thresholds were backwards. Original: score < 80 = SAFE, < 90 = SUSPICIOUS, else MALICIOUS. This means nearly all URLs were SAFE.
Fix: score 0-15 = SAFE, 16-45 = SUSPICIOUS, 46+ = MALICIOUS.
BUG 10
Confidence score was hardcoded to 100. Every URL returned 100% confidence regardless of how many signals fired, making the score meaningless.
Fix: Confidence = 40 + (signalCount × 10), capped at 95. Zero signals = 60% (baseline uncertainty).
